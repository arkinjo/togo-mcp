What is TogoMCP?

TogoMCP is a comprehensive Model Context Protocol (MCP) server that provides LLM agents with access to a vast ecosystem of life sciences databases. It integrates over 20 major biological and biomedical databases, offering researchers a powerful toolkit for cross-database queries, data integration, and knowledge discovery.

Key Features

  • Multi-Database Integration: Access to 20+ curated databases including UniProt, PubChem, PDB, ChEMBL, Ensembl, PubMed, and more
  • SPARQL Query Support: Execute powerful SPARQL queries across RDF datasets with built-in query examples and schemas
  • ID Conversion: Seamlessly convert identifiers across different databases using TogoID integration
  • Database Search: Search for entities across multiple databases (ChEMBL, MeSH, PDB, UniProt)
  • Rich Metadata: Access detailed information about each database, including MIE files with ShEx schemas and example queries
  • Streamable HTTP Transport: Efficient communication using modern HTTP streaming protocols

Who Should Use TogoMCP?

  • Biologists and Medical Researchers: Explore relationships between genes, proteins, diseases, and drugs without writing complex queries
  • Bioinformaticians: Leverage AI to construct and optimize SPARQL queries for data analysis
  • Drug Discovery Scientists: Analyze compound-target-disease relationships across integrated databases
  • Data Scientists: Perform cross-database analysis and knowledge discovery through natural language

Usage Examples

Here are real-world examples of how TogoMCP can be used to perform complex research tasks:

Example 1: Cardiovascular Drug Target Analysis

Prompt: Find proteins that are associated with both cardiovascular diseases and have known small molecule inhibitors in ChEMBL, and classify them according to disease and drug availability.

Response:

The system architected a multi-database integration strategy, first identifying cardiovascular disease-related MeSH terms, then searching ChEMBL for relevant drug targets. Using SPARQL queries across ChEMBL's RDF database, it discovered 75+ human proteins with known inhibitors, organized by therapeutic area.

Tools Used:
  • list_databases - Identified available databases
  • search_mesh_entity - Found cardiovascular disease terms
  • search_chembl_target - Located protein targets
  • get_MIE_file - Retrieved ChEMBL schema and examples
  • run_sparql - Executed multiple queries to gather bioactivity data

Key Results:

  • 15+ validated targets for Hypertension with approved drugs
  • 20+ targets for Coronary Artery Disease (approved + developmental)
  • Sodium Channels: 57 inhibitors identified
  • L-Type Calcium Channels: 27 inhibitors identified
  • Beta-1 Adrenergic Receptor: 25+ inhibitors identified

Example 2: Human Enzyme Classification

Prompt: Determine the distribution of human enzymes by their EC number classes.

Response:

The system queried UniProt's Swiss-Prot database to retrieve human enzyme classifications. It then aggregated the data by EC (Enzyme Commission) number classes to determine the distribution pattern.

Tools Used:
  • get_MIE_file - Retrieved UniProt schema
  • run_sparql - Executed queries to extract enzyme data with EC numbers

Key Results: Analysis of 4,442 unique human enzymes revealed:

  • EC 2 (Transferases): 1,823 proteins (41.0%) - Most abundant
  • EC 3 (Hydrolases): 1,655 proteins (37.3%) - Second most abundant
  • EC 1 (Oxidoreductases): 545 proteins (12.3%)
  • EC 4 (Lyases): 151 proteins (3.4%)
  • EC 5 (Isomerases): 148 proteins (3.3%)
  • EC 6 (Ligases): 124 proteins (2.8%)
  • EC 7 (Translocases): 103 proteins (2.3%)

Example 3: Natural Products vs Synthetic Compounds

Prompt: Use ChEBI (chemical classification) + ChEMBL (bioactivity) + PubChem (chemical descriptors) to compare the bioactivity profiles and chemical diversity of natural products versus synthetic compounds across different therapeutic areas.

Response:

The system conducted a comprehensive multi-database analysis, querying ChEBI for chemical classifications, ChEMBL for bioactivity data, and PubChem for molecular descriptors. It compared natural products (e.g., Vinblastine, Paclitaxel) against synthetic drugs (e.g., Imatinib, Omeprazole).

Tools Used:
  • get_MIE_file - Retrieved schemas for ChEBI, ChEMBL, and PubChem
  • run_sparql - Executed cross-database queries
  • get_compound_attributes_from_pubchem - Retrieved molecular descriptors

Key Results:

Natural Products:

  • Ultra-high potency: 70% show IC50 < 100 nM
  • Higher chemical complexity: Average 8 chiral centers, MW 400-900 Da
  • Polypharmacology: Hit 8.5 targets on average
  • Therapeutic dominance: ~60% of anticancer drugs are natural product-derived

Synthetic Compounds:

  • Selective potency: 40% show IC50 < 100 nM, but highly target-specific
  • Simpler structures: Average 1 chiral center, MW 300-600 Da
  • Monopharmacology: Hit 2-3 targets (designed selectivity)
  • Broader coverage: Dominant in cardiovascular, CNS, metabolic diseases

Setup Guide

TogoMCP is available as a remote MCP server and can be integrated with various AI platforms. Choose your platform below:

Claude Desktop

TogoMCP can be added to Claude Desktop (available on Pro, Max, Team, and Enterprise plans) as a custom connector.

Steps:

  1. Open Claude Desktop or visit claude.ai
  2. Navigate to Settings > Connectors
  3. Scroll down and click "Add custom connector"
  4. Enter the server URL: 
    https://togomcp.rdfportal.org/mcp
  5. Click "Add" to complete the setup
  6. Enable the connector in your chat by clicking the "Search and tools" button

Note: For Team and Enterprise plans, Primary Owners or Owners must first enable the connector in Admin settings before individual users can connect.

Learn more: Getting Started with Custom Connectors

ChatGPT (Developer Mode)

ChatGPT supports TogoMCP through Developer Mode (available on Business, Enterprise, and Edu plans).

Steps:

  1. Open ChatGPT and go to Settings
  2. Navigate to Connectors > Advanced
  3. Toggle on "Developer Mode"
  4. Return to the Connectors tab and click "Create"
  5. Fill in the connector details:
    • Name: TogoMCP
    • Description: Life sciences database integration via MCP
    • Server URL: https://togomcp.rdfportal.org/mcp
    • Authentication: No authentication
  6. Check "I trust this application" and click "Create"
  7. In a chat, click the + button, select More > Developer Mode
  8. Enable the TogoMCP connector to start using it

Important: TogoMCP uses Streamable HTTP transport, not SSE.

Learn more: ChatGPT Developer Mode Guide

Gemini CLI

Gemini CLI supports MCP servers through configuration in the settings.json file.

Steps:

  1. Locate your Gemini CLI settings file:
    • macOS/Linux: ~/.config/gemini-cli/settings.json
    • Windows: %APPDATA%\gemini-cli\settings.json
  2. Add TogoMCP to the mcpServers section: 
    {
  "mcpServers": {
    "togomcp": {
      "httpUrl": "https://togomcp.rdfportal.org/mcp"
    }
  }
}
  3. Save the file and restart Gemini CLI
  4. Verify the connection by running: 
    gemini mcp list
  5. You should see togomcp in the list of connected servers

Important: TogoMCP uses Streamable HTTP transport. Use "httpUrl" to specify the endpoint URL.

Learn more: MCP Servers with Gemini CLI

Available Databases

TogoMCP provides access to 22 curated life sciences databases spanning proteins, chemicals, genomics, diseases, and literature:

UniProt

Comprehensive protein sequence and functional information. Contains 444M proteins with Swiss-Prot (manually curated, 923K) and TrEMBL (automatically annotated, 444M). Cross-references to 200+ databases.

PubChem

Comprehensive chemical database with 119M compounds, 339M substances, 1.7M bioassays, 167K genes, 249K proteins, and 81K pathways. Central hub linking compounds to biological data.

PDB (Protein Data Bank)

3D structural data for 204K+ proteins, nucleic acids, and complexes from X-ray, NMR, and cryo-EM. Includes experimental methods, resolution data, and cross-references to UniProt and EMDB.

ChEMBL

Bioactive molecules database with 2.4M+ compounds, 1.6M assays, 20M bioactivity measurements. Essential for drug discovery with compound-target-activity relationships.

ChEBI

Chemical Entities of Biological Interest ontology with 217,000+ small molecules, atoms, ions, and functional groups with hierarchical classification and biological roles.

Reactome

Pathway database with 22,000+ pathways across 30+ species, molecular interactions, biochemical reactions, protein complexes, and disease associations based on BioPAX Level 3.

Rhea

Biochemical reactions database with 17,078 reactions, 11,763 compounds, all atom-balanced and linked to ChEBI. Reference resource for UniProtKB enzyme annotation.

Ensembl

Genome annotations for 100+ species with genes, transcripts, proteins, and exons. Cross-referenced to UniProt, HGNC, NCBI Gene, Reactome, and OMIM.

NCBI Gene

Gene database with 57M+ entries covering protein-coding genes, ncRNAs, pseudogenes across all organisms. Includes gene symbols, chromosomal locations, and orthology relationships.

PubMed

Bibliographic database for biomedical literature with publication metadata, MeSH term annotations, and cross-references. Enables semantic querying across millions of citations.

PubTator Central

Biomedical entity annotations from PubMed literature with Disease and Gene annotations using MeSH and NCBI Gene identifiers. Enables literature-based discovery.

Gene Ontology (GO)

Controlled vocabulary with 30,804 biological process terms, 12,793 molecular function terms, and 4,568 cellular component terms for standardized gene product annotation.

MeSH

Medical Subject Headings - NLM's controlled vocabulary thesaurus with hierarchical structure of descriptors and qualifiers across 16 main categories for biomedical literature indexing.

MONDO

Monarch Disease Ontology integrating multiple disease databases (OMIM, Orphanet, DOID, MESH, ICD) into unified classification with cross-references to 35+ databases.

NANDO

Japanese intractable diseases ontology with 2,777 disease classes, multilingual labels, and cross-references to MONDO and KEGG. Supports Japanese healthcare policy and research.

ClinVar

Genomic variation database with 3.5M+ variant records, clinical interpretations, gene associations, and disease conditions. Cross-referenced to MedGen, OMIM, MeSH, and HGNC.

MedGen

Medical genetics portal with 233,000+ clinical concepts covering diseases, phenotypes, and clinical findings. Integrates OMIM, Orphanet, HPO, and MONDO data.

DDBJ

DNA Data Bank of Japan with nucleotide sequences and genomic annotations. Part of INSDC collaboration with cross-references to BioProject, BioSample, and taxonomic databases.

GlyCosmos

Comprehensive glycoscience portal integrating glycan structures (GlyTouCan), glycoproteins, glycosylation sites, glycogenes, and lectin-glycan interactions across 100+ named graphs.

NCBI Taxonomy

Biological taxonomic classification with 3M+ organisms from bacteria to mammals with hierarchical relationships, scientific/common names, and genetic code assignments.

BacDive

Bacterial Diversity Metadatabase with 97,000+ strain records covering taxonomy, morphology, physiology, cultivation conditions, and molecular data for bacteria and archaea.

MediaDive

Microbial culture media database with 3,289 standardized recipes, 1,489 ingredients with chemical cross-references, and 45,685 strain records with growth conditions.

Available Tools

TogoMCP provides a comprehensive set of tools for database exploration and data integration:

list_databases

Lists all available databases with descriptions, helping users discover which databases are accessible and understand their scope.

get_sparql_endpoints

Retrieves the SPARQL endpoint URLs for all RDF databases, enabling direct access to query services.

get_MIE_file

Gets the MIE (Metadata, Instances, and Examples) file containing ShEx schema, RDF examples, and SPARQL query templates for a specific database. Essential for understanding database structure before querying.

get_sparql_example

Retrieves example SPARQL queries for a specific database, providing templates for common query patterns.

get_graph_list

Gets the list of named graphs available in a specific RDF database, useful for understanding data organization.

run_sparql

Executes a SPARQL query on a specific RDF database and returns the results. The core tool for data retrieval and analysis.

search_chembl_molecule

Searches for ChEMBL molecules by name or keywords, returning compound IDs and basic information.

search_chembl_target

Searches for ChEMBL protein targets by name or keywords, essential for drug discovery research.

search_chembl_id_lookup

Performs general ChEMBL ID lookup across multiple entity types (compounds, targets, assays).

search_mesh_entity

Searches for MeSH (Medical Subject Headings) terms by keywords, useful for medical and disease-related queries.

search_pdb_entity

Searches PDBj entries (protein structures, chemical components, or reference molecules) by keywords, supporting structural biology research.

search_uniprot_entity

Searches for UniProt protein entries by name or keywords, returning protein IDs and descriptions.

get_pubchem_compound_id

Retrieves PubChem Compound ID from a compound name, enabling access to chemical information.

get_compound_attributes_from_pubchem

Gets comprehensive compound attributes from PubChem RDF including molecular properties, identifiers, and classifications.

togoid_convertId

Converts IDs from one database to another using TogoID service. Supports multi-step conversions across diverse biological categories.

togoid_countId

Counts how many IDs can be converted between databases, useful for understanding data overlap.

togoid_getAllDataset

Gets configuration for all available TogoID datasets including labels, ID patterns, prefixes, and examples.

togoid_getAllRelation

Gets all possible conversion relationships between databases in TogoID.

togoid_getDataset

Gets configuration for a specific TogoID dataset including validation patterns and available annotations.

togoid_getRelation

Gets relationship details between two specific databases in TogoID.

RDF_Portal_Guide

Provides general guidelines for using the RDF Portal, including best practices for SPARQL queries and data access.
