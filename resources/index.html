üìñ Overview

TogoMCP is a comprehensive Model Context Protocol (MCP) server that provides Claude with access to a vast ecosystem of life sciences databases through SPARQL queries, RDF data exploration, and ID conversion services. It integrates over 20 major biological and biomedical databases, offering researchers a powerful toolkit for cross-database queries, data integration, and knowledge discovery.

üåê Remote Server Access: TogoMCP runs as a remote MCP server at https://togomcp.rdfportal.org/mcp

Simply configure Claude Desktop to connect - no local installation required!
üéØ Key Philosophy: MIE Files ‚Üí Explore RDF ‚Üí Comprehensive SPARQL ‚Üí TogoID Only If Needed

The server emphasizes understanding graph structures through MIE (Metadata Interoperability Exchange) files before querying, exploring RDF cross-references first, and using comprehensive SPARQL queries to achieve 70-95% coverage with 2x faster performance and higher transparency.

üîÑ Recommended Workflow

The MIE-First Approach (10-20 minutes)

  1. MIE File Analysis - Understand graph structure, properties, and examples
  2. RDF Exploration - Discover available cross-references in the database
  3. Comprehensive SPARQL - Get ALL data in ONE query using OPTIONAL patterns
  4. Execution - Run query and extract cross-reference IDs
  5. TogoID (If Needed) - Only for missing databases with good coverage
  6. Final Report - Consolidate all findings with documentation
‚ö° Coverage Expectations:
  • Field-Leading: 80-95% SPARQL + 5-10% TogoID = 85-100% total
  • Publication Quality: 60-80% SPARQL + 10-20% TogoID = 70-90% total
  • Proof-of-Concept: 40-60% SPARQL + 15-25% TogoID = 55-75% total

‚öôÔ∏è Setup for Claude Desktop

üåê Remote Server: TogoMCP runs as a remote MCP server at https://togomcp.rdfportal.org/mcp
No local installation required! Simply configure Claude Desktop to connect to the remote endpoint.

1Locate Configuration File

Find your Claude Desktop configuration file at:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json

2Add Remote TogoMCP Server

Edit the configuration file and add the TogoMCP remote server to the mcpServers section:

{
  "mcpServers": {
    "TogoMCP": {
      "url": "https://togomcp.rdfportal.org/mcp"
    }
  }
}

If you already have other MCP servers configured:

{
  "mcpServers": {
    "other-server": {
      "command": "node",
      "args": ["path/to/other/server.js"]
    },
    "TogoMCP": {
      "url": "https://togomcp.rdfportal.org/mcp"
    }
  }
}

3Restart Claude Desktop

After saving the configuration file, completely quit and restart Claude Desktop for the changes to take effect.

Important: Make sure to fully quit the application (not just close the window) before restarting.

4Verify Connection

In Claude Desktop, you should see a üîå icon in the lower right corner indicating MCP servers are connected. You can verify TogoMCP is working by asking:

"Can you list the available databases in TogoMCP?"

Claude should be able to call the list_databases tool and show you all 22 available databases.

‚ö†Ô∏è Troubleshooting

  • Server not connecting? Check your internet connection and verify the URL is correct
  • Tools not appearing? Ensure you've completely quit and restarted Claude Desktop
  • JSON syntax errors? Validate your configuration file with a JSON validator
  • Multiple servers? Make sure each server has a unique name and proper comma separation

üéØ Best Practices

  • ‚úÖ Always read MIE files first Understanding the graph structure is critical for effective queries
  • ‚úÖ Explore RDF before converting Most cross-references exist in anchor RDF databases (70-95% coverage)
  • ‚úÖ Use OPTIONAL in SPARQL Don't make all cross-references REQUIRED - use OPTIONAL patterns to maximize results
  • ‚úÖ Test TogoID coverage first Use countId before full conversion to verify coverage is >50%
  • ‚ùå Don't skip MIE analysis This leads to inefficient queries and missed opportunities
  • ‚ùå Don't assume TogoID is needed Check RDF cross-references first - they're often sufficient
  • ‚ùå Don't create multiple queries Use ONE comprehensive SPARQL query with OPTIONAL patterns
